# Правила написания Git commit сообщений

## Критическая проблема: UTF-8 и кириллица в Windows

**ВАЖНО**: Git в Windows с кодировкой UTF-8 "съедает" первый символ кириллицы, если русский текст начинается сразу после префикса типа коммита.

### Примеры проблемы

❌ **НЕПРАВИЛЬНО** (первый символ будет потерян):
```
fix: Исправлена страница checkout
```
Результат в Git: `fix: справлена страница checkout` (потерян символ "И")

❌ **НЕПРАВИЛЬНО**:
```
feat: Добавлена интеграция с API
```
Результат в Git: `feat: обавлена интеграция с API` (потерян символ "Д")

✅ **ПРАВИЛЬНО** (латинский префикс защищает кириллицу):
```
fix: Fixed страница checkout
feat: Added интеграция с API
docs: Updated документация
```

## Решение проблемы

### Правило 1: Всегда начинайте с латинского слова

После `type:` или `type(scope):` ВСЕГДА ставьте латинское слово перед русским текстом:

```
fix: Fixed ошибка CartViewModel
feat: Added новая функция корзины
docs: Updated документация по checkout
refactor: Refactored метод loadPoints()
style: Improved стили для мобильных устройств
test: Added тесты для OrderService
chore: Updated зависимости проекта
```

### Правило 2: Используйте стандартные латинские глаголы

Рекомендуемые глаголы для каждого типа:

- `fix:` → **Fixed** (исправлено)
- `feat:` → **Added** (добавлено)
- `docs:` → **Updated** (обновлено)
- `refactor:` → **Refactored** (рефакторинг)
- `style:` → **Improved** / **Updated** (улучшено/обновлено)
- `test:` → **Added** / **Updated** (добавлено/обновлено)
- `chore:` → **Updated** / **Changed** (обновлено/изменено)
- `perf:` → **Optimized** (оптимизировано)

## Формат commit сообщений (Conventional Commits)

```
<type>(<scope>): <Latin_word> <Russian_description>

[optional body]

[optional footer]
```

### Типы коммитов

- **feat**: Новая функциональность
- **fix**: Исправление ошибки
- **docs**: Изменения в документации
- **style**: Форматирование, отступы (не влияет на код)
- **refactor**: Рефакторинг кода (без изменения функциональности)
- **perf**: Улучшение производительности
- **test**: Добавление или изменение тестов
- **chore**: Обновление зависимостей, конфигурации, и т.д.
- **build**: Изменения в системе сборки
- **ci**: Изменения в CI/CD конфигурации

### Scope (опционально)

Указывает область изменений:

```
feat(checkout): Added валидация формы заказа
fix(cart): Fixed ошибка подсчета суммы
docs(api): Updated документация REST API
```

## Примеры правильных commit сообщений

### Исправление ошибок

```
fix: Fixed ошибка CartViewModel is not defined
fix(checkout): Fixed проблема с загрузкой пунктов выдачи
fix(cart): Fixed некорректный подсчет количества товаров
```

### Новая функциональность

```
feat: Added интеграция с Яндекс.Картами
feat(checkout): Added выбор способа доставки
feat(payment): Added поддержка оплаты картой
```

### Документация

```
docs: Updated требования к данным карточек товаров
docs(blade): Added правила подключения скриптов
docs(checkout): Updated документация по миграции
```

### Рефакторинг

```
refactor: Refactored метод getProductRating()
refactor(controllers): Unified формат данных корзины
refactor(services): Extracted OrderService из контроллера
```

### Стили и UI

```
style: Improved стили для мобильных устройств
style(checkout): Updated дизайн формы заказа
style(cart): Fixed отступы в карточках товаров
```

### Тесты

```
test: Added тесты для OrderService
test(cart): Updated тесты корзины
test(integration): Added интеграционные тесты checkout
```

### Техническое обслуживание

```
chore: Updated зависимости Laravel
chore(config): Changed настройки кэширования
chore(deps): Bumped версия PHP до 8.2
```

## Многострочные commit сообщения

Для сложных изменений используйте body и footer:

```
feat: Added интеграция с Яндекс.Картами для выбора пунктов выдачи

Реализовано:
- Загрузка карты через Yandex Maps API
- Отображение пунктов выдачи на карте
- Выбор пункта выдачи кликом
- Сохранение выбранного пункта в сессии

Closes #123
```

**ВАЖНО**: В body также начинайте строки с латинских символов или используйте маркеры списка (`-`, `*`).

## Что НЕЛЬЗЯ делать

❌ **Начинать русский текст сразу после двоеточия**:
```
fix: Исправлена ошибка
feat: Добавлена функция
```

❌ **Использовать только русский текст**:
```
Исправлена ошибка CartViewModel
Добавлены алиасы методов
```

❌ **Писать неинформативные сообщения**:
```
fix: Fixed баг
feat: Added фича
chore: Updated файлы
```

❌ **Смешивать несколько типов изменений в одном коммите**:
```
feat: Added новая страница и fixed ошибки и updated документация
```

## Что НУЖНО делать

✅ **Всегда использовать type prefix**:
```
fix: Fixed ошибка
feat: Added функция
```

✅ **Начинать с латинского слова**:
```
fix: Fixed ошибка CartViewModel is not defined
feat: Added интеграция с Яндекс.Картами
```

✅ **Быть конкретным и информативным**:
```
fix: Fixed ошибка CartViewModel is not defined - используется model_cart из layout
feat: Added интеграция с Яндекс.Картами для выбора пунктов выдачи
```

✅ **Разделять разные типы изменений на отдельные коммиты**:
```
Коммит 1: feat: Added новая страница checkout
Коммит 2: docs: Updated документация по checkout
Коммит 3: test: Added тесты для checkout
```

## Настройка Git для корректной работы с UTF-8

Рекомендуется настроить Git для правильной работы с кириллицей:

```bash
git config --global core.quotepath false
git config --global i18n.commitencoding utf-8
git config --global i18n.logoutputencoding utf-8
```

Однако даже с этими настройками проблема может сохраняться в Windows, поэтому **всегда используйте латинский префикс**.

## Проверка перед commit

Перед выполнением `git commit` проверьте:

1. ✅ Есть ли type prefix (`fix:`, `feat:`, и т.д.)?
2. ✅ Начинается ли текст после двоеточия с латинского слова?
3. ✅ Описание информативное и конкретное?
4. ✅ Коммит содержит только один тип изменений?
