# Инструкция по проверке исправления сессии корзины

## Автоматическое тестирование

Запустите тест для проверки сохранения данных в сессию:

```bash
php artisan test tests/Feature/CartSessionPersistenceTest.php
```

или

```bash
vendor/bin/phpunit tests/Feature/CartSessionPersistenceTest.php
```

## Ручное тестирование

### 1. Проверка добавления товара через AJAX

1. Откройте страницу каталога или главную страницу
2. Откройте DevTools (F12) → вкладка Network
3. Нажмите кнопку "В корзину" на любом товаре
4. В Network найдите запрос к `/api/cart` с методом POST
5. Проверьте ответ - должен содержать:
   ```json
   {
     "items": {...},
     "total_cart_sum": ...,
     "total_cart_amount": ...
   }
   ```
6. Проверьте, что счетчик корзины в header обновился

### 2. Проверка сохранения данных после перезагрузки

1. Добавьте товар в корзину (как в шаге 1)
2. Перезагрузите страницу (F5)
3. Проверьте, что счетчик корзины показывает правильное количество товаров
4. Перейдите на страницу `/cart`
5. Убедитесь, что добавленный товар отображается в корзине

### 3. Проверка получения данных корзины

1. Откройте DevTools → Console
2. Выполните команду:
   ```javascript
   fetch('/api/cart', {
     method: 'POST',
     headers: {
       'Content-Type': 'application/x-www-form-urlencoded',
     },
     body: 'task=get_cart'
   })
   .then(r => r.json())
   .then(data => console.log(data));
   ```
3. Проверьте, что в консоли отображаются данные корзины

### 4. Проверка изменения количества

1. Перейдите на страницу `/cart`
2. Измените количество товара с помощью кнопок +/-
3. Проверьте, что количество обновляется
4. Перезагрузите страницу
5. Убедитесь, что новое количество сохранилось

### 5. Проверка удаления товара

1. Перейдите на страницу `/cart`
2. Нажмите кнопку удаления товара
3. Проверьте, что товар исчез из списка
4. Проверьте, что счетчик в header обновился
5. Перезагрузите страницу
6. Убедитесь, что товар не вернулся в корзину

## Ожидаемые результаты

✅ Все тесты проходят успешно
✅ Товары добавляются в корзину через AJAX
✅ Данные корзины сохраняются в сессию
✅ После перезагрузки страницы корзина остается заполненной
✅ Счетчики корзины обновляются корректно
✅ Изменение количества и удаление товаров работает

## Возможные проблемы

### Проблема: Тесты не запускаются

**Решение**: Убедитесь, что PHP установлен и доступен в PATH:
```bash
php --version
```

Если PHP не найден, используйте полный путь к PHP или настройте PATH.

### Проблема: Данные не сохраняются в сессию

**Решение**: Проверьте, что в `routes/api.php` маршрут `/api/cart` использует `web` middleware:
```php
Route::post('/cart', [CartController::class, 'handleAjax'])
    ->middleware(['web'])
    ->withoutMiddleware([\App\Http\Middleware\VerifyCsrfToken::class])
    ->name('api.cart');
```

### Проблема: CSRF token error

**Решение**: Убедитесь, что `withoutMiddleware([\App\Http\Middleware\VerifyCsrfToken::class])` присутствует в маршруте.

### Проблема: 404 Not Found

**Решение**: Очистите кеш маршрутов:
```bash
php artisan route:clear
php artisan cache:clear
```
