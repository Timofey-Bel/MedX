# Bugfix Requirements Document

## Introduction

После миграции с legacy системы на Laravel, функциональность корзины перестала работать. Товары не добавляются в корзину, счетчики не обновляются, страница корзины остается пустой. Проблема связана с отсутствием интеграции Knockout.js модели корзины из legacy системы (`public/js/models/cart_new/`) в новую Laravel-архитектуру.

В legacy системе корзина работала через модульную Knockout.js модель с AJAX запросами к `/cart/` endpoint. В новой системе:
- Модель корзины не подключена в layouts
- AJAX endpoints не реализованы в CartController
- Глобальная переменная `server_cart.data` не инициализирована
- Счетчики в header не привязаны к Knockout.js модели
- Кнопки "В корзину" не имеют обработчиков событий

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN пользователь кликает на кнопку "В корзину" на карточке товара THEN система не добавляет товар в корзину и не показывает никакой реакции

1.2 WHEN пользователь добавляет товар в корзину THEN счетчик товаров в header не обновляется и остается на нулевом значении

1.3 WHEN пользователь переходит на страницу /cart после добавления товара THEN страница показывает "Корзина пуста" вместо списка добавленных товаров

1.4 WHEN пользователь кликает на кнопку "В корзину" THEN анимация добавления товара (красный шарик летит к иконке корзины) не воспроизводится

1.5 WHEN пользователь пытается изменить количество товара через кнопки +/- на странице корзины THEN количество не изменяется и нет реакции на клики

1.6 WHEN пользователь пытается удалить товар из корзины THEN товар не удаляется и остается в списке

1.7 WHEN страница загружается THEN глобальная переменная `server_cart.data` не инициализирована и Knockout.js модель корзины не доступна

1.8 WHEN выполняется AJAX запрос к `/cart/` с параметром `task` (put_item, delete_product, update_amount, get_cart) THEN сервер возвращает ошибку 404 или 500, так как endpoints не реализованы

### Expected Behavior (Correct)

2.1 WHEN пользователь кликает на кнопку "В корзину" на карточке товара THEN система SHALL добавить товар в корзину, обновить счетчик и показать визуальную обратную связь

2.2 WHEN пользователь добавляет товар в корзину THEN счетчик товаров в header SHALL обновиться и показать актуальное количество товаров в корзине

2.3 WHEN пользователь переходит на страницу /cart после добавления товара THEN страница SHALL отобразить список всех добавленных товаров с их названиями, ценами и количеством

2.4 WHEN пользователь кликает на кнопку "В корзину" THEN анимация добавления товара (красный шарик летит к иконке корзины) SHALL воспроизвестись

2.5 WHEN пользователь кликает на кнопки +/- на странице корзины THEN количество товара SHALL изменяться соответственно и обновляться на сервере

2.6 WHEN пользователь кликает на кнопку удаления товара из корзины THEN товар SHALL быть удален из корзины и исчезнуть из списка

2.7 WHEN страница загружается THEN глобальная переменная `server_cart.data` SHALL быть инициализирована с текущим состоянием корзины из сессии Laravel

2.8 WHEN выполняется AJAX запрос к `/cart/` с параметром `task` (put_item, delete_product, update_amount, get_cart) THEN сервер SHALL обработать запрос и вернуть корректный JSON ответ с обновленным состоянием корзины

### Unchanged Behavior (Regression Prevention)

3.1 WHEN пользователь просматривает каталог товаров без взаимодействия с корзиной THEN система SHALL CONTINUE TO отображать товары корректно без изменений в производительности

3.2 WHEN пользователь использует другие функции сайта (поиск, фильтры, навигация) THEN система SHALL CONTINUE TO работать без влияния на функциональность корзины

3.3 WHEN пользователь работает с корзиной в legacy части системы (если она еще существует) THEN система SHALL CONTINUE TO использовать ту же структуру данных корзины в сессии

3.4 WHEN загружаются страницы без товаров (статические страницы, профиль) THEN система SHALL CONTINUE TO работать без необходимости инициализации модели корзины

3.5 WHEN пользователь работает с избранным (favorites) или другими списками товаров THEN система SHALL CONTINUE TO работать независимо от состояния корзины

3.6 WHEN выполняются другие AJAX запросы к серверу (не связанные с корзиной) THEN система SHALL CONTINUE TO обрабатывать их без конфликтов с новыми cart endpoints

3.7 WHEN используется существующая структура Knockout.js модели из `public/js/models/cart_new/` THEN система SHALL CONTINUE TO использовать те же методы и свойства модели для обратной совместимости
