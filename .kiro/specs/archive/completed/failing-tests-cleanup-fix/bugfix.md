# Bugfix Requirements Document

## Introduction

При запуске полного набора тестов командой `php vendor/bin/phpunit --colors=always` обнаружено массовое падение тестов:
- Tests: 90
- Assertions: 209  
- Errors: 33
- Failures: 30
- PHPUnit Deprecations: 50

Итого: 63 теста падают (33 ошибки + 30 failures) + 50 предупреждений о deprecation.

Проект находится в процессе миграции с legacy Smarty системы на Laravel 12. Недавно были созданы новые тесты для корзины и страницы товара, которые должны работать корректно. Однако существуют старые тесты, связанные с CSRF и сессиями корзины, которые могли устареть из-за изменений в архитектуре.

Проблема влияет на:
- Надежность CI/CD процесса
- Уверенность в качестве кода
- Возможность рефакторинга без страха сломать функциональность
- Время разработки (невозможно отличить реальные баги от устаревших тестов)

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN запускается полный набор тестов THEN 63 теста падают с ошибками и failures

1.2 WHEN выполняются тесты THEN генерируется 50 PHPUnit deprecation warnings

1.3 WHEN присутствуют Laravel default тесты (ExampleTest.php в Feature и Unit) THEN они могут давать ложные срабатывания или быть неактуальными

1.4 WHEN выполняются старые CSRF-тесты (CartCsrfExceptionTest, CartCsrfExemptionTest) THEN они могут не соответствовать текущей архитектуре API

1.5 WHEN выполняются тесты сессий корзины (CartSessionPersistenceTest) THEN они могут проверять устаревшую логику работы с сессиями

1.6 WHEN новые тесты (CartFunctionalityTest - 29 тестов, ProductPageTest - 15 тестов, ProductServiceTest - 15 тестов, ProductControllerTest - 5 тестов) падают THEN это указывает на реальные проблемы в коде

### Expected Behavior (Correct)

2.1 WHEN запускается полный набор тестов THEN все актуальные тесты должны проходить успешно (0 failures, 0 errors)

2.2 WHEN выполняются тесты THEN НЕ должно быть PHPUnit deprecation warnings

2.3 WHEN присутствуют неактуальные Laravel default тесты THEN они должны быть удалены из тестового набора

2.4 WHEN старые CSRF-тесты не соответствуют текущей архитектуре THEN они должны быть обновлены или удалены

2.5 WHEN тесты сессий корзины проверяют устаревшую логику THEN они должны быть обновлены под текущую реализацию или удалены

2.6 WHEN новые тесты падают из-за реальных проблем в коде THEN код должен быть исправлен для прохождения тестов

2.7 WHEN новые тесты падают из-за неправильной настройки тестового окружения THEN настройки должны быть исправлены

### Unchanged Behavior (Regression Prevention)

3.1 WHEN новые тесты корректны и проходят THEN они должны продолжать проходить после исправлений

3.2 WHEN существует рабочая функциональность корзины THEN она должна продолжать работать после исправления тестов

3.3 WHEN существует рабочая функциональность страницы товара THEN она должна продолжать работать после исправления тестов

3.4 WHEN тесты используют реальную базу данных (согласно phpunit.xml) THEN это поведение должно сохраниться

3.5 WHEN тесты проверяют интеграцию с Knockout.js THEN эта функциональность должна продолжать работать корректно
