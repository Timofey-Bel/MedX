# Команда sections:wrap-css

## Назначение

Команда `sections:wrap-css` автоматически добавляет префикс `[data-section-guid="..."]` ко всем CSS селекторам в существующих секциях из таблицы `page_sections`. Это обеспечивает изоляцию стилей секций от остальной страницы.

## Использование

### Просмотр изменений без применения (dry-run)

```bash
php artisan sections:wrap-css --dry-run
```

Показывает, какие изменения будут применены, но не изменяет данные в БД.

### Применение изменений

```bash
php artisan sections:wrap-css
```

Обновляет CSS во всех активных секциях, добавляя префиксы к селекторам.

### Обработка конкретной секции

```bash
php artisan sections:wrap-css --guid=kawsfk9k
```

Обрабатывает только секцию с указанным GUID.

## Что делает команда

1. **Находит секции**: Получает все активные секции с непустым полем `css`
2. **Проверяет префикс**: Пропускает секции, где CSS уже содержит правильный префикс
3. **Добавляет префикс**: Автоматически добавляет `[data-section-guid="GUID"]` к каждому CSS селектору
4. **Обновляет БД**: Сохраняет обработанный CSS в таблицу `page_sections`
5. **Очищает кеш**: Удаляет кеш секции для немедленного применения изменений

## Пример обработки

### До обработки

```css
.hero-section {
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    padding: 60px 20px;
}

.hero-section h1 {
    font-size: 48px;
    color: white;
}
```

### После обработки

```css
[data-section-guid="kawsfk9k"] .hero-section {
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    padding: 60px 20px;
}

[data-section-guid="kawsfk9k"] .hero-section h1 {
    font-size: 48px;
    color: white;
}
```

## Особенности обработки

### Что обрабатывается

- ✅ Обычные селекторы классов (`.class`)
- ✅ Селекторы элементов (`div`, `p`, `h1`)
- ✅ Комбинированные селекторы (`.class1 .class2`)
- ✅ Псевдоклассы (`.class:hover`)

### Что НЕ обрабатывается (остается без изменений)

- ⏭️ At-rules (`@media`, `@keyframes`, `@import`)
- ⏭️ Комментарии (`/* ... */`, `// ...`)
- ⏭️ Пустые строки
- ⏭️ CSS, который уже содержит правильный префикс

## Когда использовать

### При миграции существующих секций

Если в БД уже есть секции с CSS без префиксов, запустите команду один раз для обработки всех секций:

```bash
php artisan sections:wrap-css
```

### При импорте секций из другой системы

После импорта секций из legacy-системы или другого источника:

```bash
php artisan sections:wrap-css
```

### При обновлении конкретной секции

Если вручную обновили CSS секции в админ-панели и забыли добавить префиксы:

```bash
php artisan sections:wrap-css --guid=kawsfk9k
```

## Безопасность

- ✅ Команда работает только с активными секциями (`active = 1`)
- ✅ Не изменяет секции, где префикс уже есть
- ✅ Автоматически очищает кеш после обновления
- ✅ Поддерживает dry-run для предварительного просмотра
- ✅ Транзакционная обработка (если одна секция не обработается, остальные все равно обновятся)

## Ограничения

### Простая эвристика

Команда использует простую построчную обработку CSS. Это означает:

- ❌ Не поддерживает многострочные селекторы
- ❌ Может некорректно обработать сложные вложенные at-rules
- ❌ Не понимает SCSS/LESS синтаксис

Для production-окружения с очень сложным CSS рекомендуется использовать полноценный CSS-парсер.

### Рекомендации для редакторов контента

После обработки команды, при создании новых секций в админ-панели, редакторы должны сразу писать CSS с префиксами:

```css
/* ПРАВИЛЬНО */
[data-section-guid="my-guid"] .my-class {
    color: red;
}

/* НЕПРАВИЛЬНО - потребуется повторный запуск команды */
.my-class {
    color: red;
}
```
