# Миграция проекта "Творческий Центр СФЕРА" на Laravel 12

Этот документ описывает процесс миграции легаси PHP-приложения, использующего Smarty, на современный фреймворк Laravel 12.

## 1. Обзор проекта и цель миграции

**Исходный проект:** Легаси PHP-приложение, разработанное на кастомном движке с использованием шаблонизатора Smarty.
**Цель миграции:** Перевести функционал приложения на Laravel 12, сохранив существующую базу данных и по возможности используя `DB::select` для сложных запросов, а также поддерживая модульность JavaScript с помощью Blade-шаблонов.

## 2. Выполненная работа

### 2.1. Настройка базы данных и моделей

*   **База данных:** Импортирован дамп существующей базы данных.
*   **Подключение:** Настроено подключение Laravel к базе данных через `.env`.
*   **Модели Eloquent:** Сгенерированы Eloquent-модели для всех существующих таблиц базы данных.

### 2.2. Миграция конфигурации

*   **`config.php` -> `.env`:** Ключевые константы из легаси `config.php` перенесены в файл `.env` Laravel.
*   **`config/custom.php`:** Создан новый конфигурационный файл `config/custom.php` для структурированного доступа к пользовательским настройкам.

### 2.3. Миграция маршрутизации

*   **`web.php`:**
    *   Настроены основные маршруты для главной страницы, каталога, страницы продукта.
    *   Реализованы чистые URL для категорий (`/catalog/{category_id}`).
    *   Временно закомментированы маршруты `/cart` и `/favorites` для устранения конфликтов.
*   **`SitemapController.php`:** Создан контроллер для генерации `sitemap.xml` и добавлен соответствующий маршрут.

### 2.4. Миграция модуля "Каталог"

*   **`CatalogController.php`:**
    *   Создан контроллер для обработки логики каталога.
    *   Методы `getProducts` и `getProductsCount` переписаны с использованием `DB::select` для сохранения возможности выполнения сложных SQL-запросов.
    *   Реализована логика получения URL изображений продуктов и их рейтинга.
    *   Интегрирована функция `get_subcategories_ids` для корректной фильтрации по категориям, включая подкатегории.
    *   Интегрирован `FilterService` для обработки фильтров.
*   **`resources/views/catalog/index.blade.php`:**
    *   Легаси шаблон `catalog.tpl` конвертирован в Blade-шаблон.
    *   Реализовано отображение списка категорий в сайдбаре.
    *   Добавлен фильтр по авторам с использованием `FilterService`.
    *   Добавлена форма для применения фильтров.
*   **`resources/views/catalog/partials/category_item.blade.php`:** Создан частичный шаблон для рекурсивного отображения категорий.
*   **`app/Services/FilterService.php`:** Создан сервис для инкапсуляции логики фильтрации (реализован `getAuthors`, `getAges` временно отложен).

### 2.5. Миграция модуля "Страница продукта"

*   **`ProductController.php`:**
    *   Создан контроллер для обработки логики страницы продукта.
    *   Портирована логика получения данных продукта, хлебных крошек, атрибутов, изображений, отзывов из легаси `product.class.php`.
    *   Исправлен путь к изображению-заглушке (`product_empty.jpg`).
*   **`resources/views/product/show.blade.php`:**
    *   Легаси шаблон `product.tpl` конвертирован в Blade-шаблон.
    *   Добавлен мета-тег `csrf-token` для безопасности AJAX-запросов.
    *   Включены Blade-шаблоны для общих элементов (`favicon`, `scripts`, `header`, `mobile_menu`).
*   **Статические ресурсы:**
    *   Выявлены и скопированы необходимые статические файлы (jQuery, `cart-counter.js`, `favorites-counter.js`, логотипы, фавиконы, `product_empty.jpg`, KnockoutJS библиотеки) из легаси в директорию `public/`.
    *   Обновлены пути к статическим ресурсам в Blade-шаблонах.
*   **Модульный JavaScript (KnockoutJS):**
    *   Принято решение сохранить модульность JavaScript, используя Blade-шаблоны для включения JS-файлов.
    *   Конвертированы `model_cart.js.tpl` и его включаемые файлы (`put_into_cart.js`, `remove_from_cart.js` и т.д.) в Blade-шаблоны (`resources/views/js/models/cart_new/*.blade.php`).
    *   Конвертирован `model_favorites.js.tpl` в Blade-шаблон (`resources/views/js/models/favorites/model_favorites.blade.php`).
    *   Обновлен `resources/views/sfera/tpl/scripts.blade.php` для использования Blade-включений для этих модульных JS-файлов.

### 2.6. Настройка API для счетчиков корзины/избранного

*   **`routes/api.php`:**
    *   Созданы `POST` маршруты `/api/cart` и `/api/favorites` для обработки AJAX-запросов от `cart-counter.js` и `favorites-counter.js`.
    *   Маршруты явно обернуты в группу с префиксом `api` и middleware `api` для корректной регистрации.
*   **`app/Services/CartService.php`:** Реализован базовый функционал `getCart()` для получения данных корзины из сессии.
*   **`app/Services/FavoriteService.php`:** Реализован базовый функционал `getFavorites()` для получения данных избранного из сессии.
*   **`cart-counter.js` / `favorites-counter.js`:** Обновлены URL AJAX-запросов на `/api/cart` и `/api/favorites` соответственно, а также добавлена передача `task` в теле запроса и `X-CSRF-TOKEN` в заголовках.

## 3. Текущий статус и нерешенные проблемы

### 3.1. Критическая проблема: Отсутствие `RouteServiceProvider.php`

*   **Проблема:** Файл `c:\OS\home\myapp\app\Providers\RouteServiceProvider.php`, который является фундаментальным для маршрутизации Laravel, не найден в проекте.
*   **Влияние:** Это является основной причиной проблем с регистрацией `POST` маршрутов в `api.php`, что приводит к ошибкам `405 Method Not Allowed` и возврату HTML-страниц вместо JSON для API-запросов.
*   **Наблюдение:** В директории `app/Providers` присутствует `NativeAppServiceProvider.php`, который не является стандартным.

### 3.2. Ошибки AJAX для корзины и избранного

*   **Проблема:** Несмотря на все попытки, `POST` запросы к `/api/cart` и `/api/favorites` по-прежнему возвращают `405 Method Not Allowed` и HTML-страницу.
*   **Причина:** Напрямую связана с отсутствием или некорректной работой `RouteServiceProvider.php`.
