# Реализация контентных страниц

## Обзор

Реализован универсальный модуль контентных страниц для отображения динамических страниц сайта из таблицы `pages`.

**Дата реализации:** 2026-02-27  
**Ветка:** feature/content-pages  
**Задача:** 10. Реализовать контроллер контентных страниц (Этап 21)

## Ключевой принцип

**Страницы полностью динамические** - они создаются через добавление записей в таблицу `pages`, а не через hardcode в контроллере. Контроллер универсальный и работает с любой страницей из БД.

## Реализованные компоненты

### 1. PageController

**Файл:** `app/Http/Controllers/PageController.php`

**Единственный метод:**
- `show($slug)` - универсальное отображение страницы по slug (name или id)
  - Поддерживает поиск по системному имени (name) или ID
  - Проверяет активность страницы (active = 1)
  - Возвращает 404 если страница не найдена
  - Передает в view: page, title, cart, favorites
  - Извлекает все поля: id, name, title, parent_id, content, active, sort

**Никаких hardcoded методов!** Все страницы создаются динамически через БД.

### 2. Blade шаблон

**Файл:** `resources/views/page/show.blade.php`

**Особенности:**
- Конвертирован из legacy: `site/modules/sfera/page/page.tpl`
- Использует layout с header и footer
- Отображает контент страницы через `{!! $page->content !!}`
- Подключает необходимые CSS и JS файлы

### 3. Роуты

**Файл:** `routes/web.php`

**Единственный универсальный маршрут:**

```php
// Статические страницы (универсальный роут — ДОЛЖЕН БЫТЬ ПОСЛЕДНИМ!)
// Все страницы создаются динамически через добавление записей в таблицу pages
// Примеры: /page/about, /page/contacts, /page/delivery, /page/13 (по ID)
Route::get('/page/{slug}', [PageController::class, 'show'])->name('page');
```

**Примеры использования:**
- `/page/О%20нас` - по системному имени (name)
- `/page/13` - по ID
- `/page/contacts` - по любому slug из БД

## Структура базы данных

### Таблица pages

```sql
CREATE TABLE pages (
    id INT(11) NOT NULL AUTO_INCREMENT,
    active TINYINT(1) DEFAULT 1 COMMENT '1 = активна, 0 = неактивна',
    name VARCHAR(255) NOT NULL COMMENT 'Системное имя (slug)',
    title VARCHAR(255) DEFAULT NULL COMMENT 'Заголовок страницы',
    parent_id INT(11) DEFAULT 0 COMMENT 'ID родительской страницы',
    content TEXT DEFAULT NULL COMMENT 'Содержимое страницы (HTML)',
    sort INT(11) DEFAULT 0 COMMENT 'Порядок сортировки',
    created_at TIMESTAMP NULL DEFAULT current_timestamp,
    updated_at TIMESTAMP NULL DEFAULT current_timestamp ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id)
);
```

**Поля:**
- `id` - уникальный идентификатор страницы
- `active` - флаг активности (1 = активна, 0 = скрыта)
- `name` - системное имя страницы (slug)
- `title` - заголовок страницы
- `parent_id` - ID родительской страницы (для иерархии разделов)
- `content` - HTML-контент страницы
- `sort` - порядок сортировки
- `created_at` / `updated_at` - временные метки

**SQL-запрос в контроллере:**

```sql
SELECT id, name, title, parent_id, content, active, sort
FROM pages
WHERE (name = ? OR id = ?) AND active = 1
LIMIT 1
```

## Как добавить новую страницу

Просто добавьте запись в таблицу `pages`:

```sql
INSERT INTO pages (active, name, title, parent_id, content, sort)
VALUES (
    1,
    'new-page',
    'Новая страница',
    0,
    '<h1>Содержимое новой страницы</h1><p>Текст...</p>',
    100
);
```

Страница сразу станет доступна по адресу: `/page/new-page`

## Примечания

- Роут `/page/{slug}` должен быть последним в routes/web.php, чтобы не перехватывать другие маршруты
- Поле `parent_id` позволяет создавать иерархию страниц (разделы и подразделы)
- Поле `sort` позволяет управлять порядком отображения в меню
- Все комментарии в коде на русском языке
- Контроллер максимально простой - только извлечение из БД и отображение
