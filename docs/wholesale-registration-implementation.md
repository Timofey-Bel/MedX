## Как использовать

### Регистрация физического лица (розничный покупатель)

1. Открыть `/register`
2. Выбрать "Физическое лицо"
3. Заполнить стандартные поля
4. Нажать "Зарегистрироваться"

### Регистрация юридического лица (оптовый покупатель)

1. Открыть `/register`
2. Выбрать "Юридическое лицо"
3. Заполнить стандартные поля
4. Ввести ИНН организации
5. Нажать "Проверить ИНН"
6. Данные организации загрузятся автоматически
7. При необходимости отредактировать незаблокированные поля
8. Нажать "Зарегистрироваться"

## API Endpoints

### POST /api/dadata/check-inn

Проверка ИНН организации через DaData.

**Request:**
```json
{
  "inn": "7707083893"
}
```

**Response (успех):**
```json
{
  "success": true,
  "message": "Организация найдена",
  "data": {
    "inn": "7707083893",
    "kpp": "773601001",
    "ogrn": "1027700132195",
    "name_full": "ПУБЛИЧНОЕ АКЦИОНЕРНОЕ ОБЩЕСТВО \"СБЕРБАНК РОССИИ\"",
    "name_short": "ПАО СБЕРБАНК",
    "legal_address": "г Москва, ул Вавилова, д 19",
    "director_name": "Греф Герман Оскарович",
    "director_position": "ПРЕЗИДЕНТ, ПРЕДСЕДАТЕЛЬ ПРАВЛЕНИЯ",
    "opf": "Публичное акционерное общество",
    "status": "ACTIVE"
  }
}
```

**Response (ошибка):**
```json
{
  "success": false,
  "message": "Организация с таким ИНН не найдена"
}
```

## Технические детали

### Валидация ИНН

Реализована проверка контрольной суммы для:
- ИНН из 10 цифр (юридические лица)
- ИНН из 12 цифр (физические лица-предприниматели)

### Кеширование

Результаты проверки ИНН кешируются на 24 часа для снижения нагрузки на API DaData.

### Безопасность

- CSRF защита для всех форм
- Валидация на стороне сервера
- Транзакции для целостности данных
- Rate limiting для API (встроенный в Laravel)

### Обработка ошибок

- Проверка статуса организации (ликвидирована/активна)
- Обработка ошибок API DaData
- Откат транзакций при ошибках
- Понятные сообщения об ошибках для пользователя

## Примечания

- Все изменения в базе данных выполнены через SQL (без миграций)
- Токен DaData уже настроен в `.env`
- Полный JSON ответ от DaData сохраняется для возможного использования в будущем
- Поддерживается обновление существующих организаций при повторной регистрации
