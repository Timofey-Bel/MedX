# Функционал поиска товаров

## Обзор

Модуль поиска позволяет пользователям искать товары по названию или ID товара. Включает основную страницу результатов поиска и AJAX автодополнение в header.

## Файлы

### Backend
- `app/Http/Controllers/SearchController.php` - контроллер поиска
- `routes/web.php` - маршруты `/search` и `/api/search/autocomplete`

### Frontend
- `resources/views/search/index.blade.php` - страница результатов поиска
- `public/assets/sfera/js/header.js` - JavaScript для автодополнения
- `public/assets/sfera/css/search.css` - стили для автодополнения

## Основные возможности

### 1. Страница результатов поиска

**URL**: `/search?query={поисковый_запрос}`

**Функционал**:
- Поиск по названию товара (LIKE '%query%')
- Поиск по ID товара (LIKE '%query%')
- Пагинация (20 товаров на страницу)
- Отображение карточек товаров с:
  - Изображением
  - Названием
  - Ценой
  - Рейтингом и количеством отзывов
  - Кнопками "В корзину" и "Купить всё"
  - Кнопкой добавления в избранное
- Сохранение поискового запроса в сессии
- Breadcrumbs навигация

**Контроллер**: `SearchController::index()`

**Методы**:
- `getProducts()` - получение товаров с фильтрами и пагинацией
- `getProductsCount()` - подсчет общего количества товаров
- `getProductImageUrl()` - получение URL изображения товара
- `getProductRating()` - получение рейтинга и количества отзывов

### 2. Автодополнение поиска (AJAX)

**Endpoint**: `/api/search/autocomplete?query={запрос}`

**Функционал**:
- Минимум 2 символа для начала поиска
- Максимум 10 подсказок
- Поиск по названию и ID товара
- Отображение:
  - Изображения товара
  - Названия
  - Цены
- Клик по подсказке переходит на страницу товара

**Контроллер**: `SearchController::autocomplete()`

**Формат ответа**:
```json
{
  "suggestions": [
    {
      "id": "00-00006779",
      "name": "Название товара",
      "price": 500,
      "url": "/product/00-00006779/"
    }
  ]
}
```

## Интеграция с корзиной и избранным

Страница результатов поиска полностью интегрирована с функционалом корзины и избранного:

1. **Корзина**: Карточки товаров показывают текущее количество в корзине
2. **Избранное**: Кнопка избранного отображает состояние (добавлено/не добавлено)
3. **Преобразование формата**: Контроллер преобразует формат избранного для неавторизованных пользователей

См. `.kiro/steering/product-card-data-requirements.md` для деталей.

## Структура данных товара

Каждый товар в результатах поиска содержит:

```php
[
    'id' => string,              // ID товара (offer_id)
    'name' => string,            // Название товара
    'description' => string,     // Описание товара
    'image' => string,           // URL изображения товара
    'price' => int,              // Цена товара (округленная)
    'quantity' => int,           // Количество товара на складе
    'rating' => float,           // Средний рейтинг (0 если нет отзывов)
    'reviews_count' => int       // Количество отзывов (0 если нет)
]
```

## Фильтрация (будущее развитие)

В текущей версии фильтрация по авторам и возрастам подготовлена в коде, но не отображается в UI. Для активации:

1. Раскомментировать секции фильтров в `resources/views/search/index.blade.php`
2. Добавить методы получения списка авторов и возрастов (как в CatalogController)
3. Передать данные фильтров в view

## Пагинация

- 20 товаров на страницу
- Группы по 5 страниц
- Кнопки "Предыдущая группа" и "Следующая группа"
- Активная страница выделена

## Сессия

Поисковый запрос сохраняется в сессии:
- Ключ: `search_query`
- Используется для восстановления запроса при возврате на страницу

## Миграция из legacy

Модуль мигрирован из `legacy/site/modules/sfera/search/search.class.php`

**Основные изменения**:
- `$_SESSION['search']` → `session(['search_query' => $query])`
- `rows()` → `DB::table()` и Query Builder
- `$_SESSION['smarty']->assign()` → `return view()`
- Прямые SQL запросы → Laravel Query Builder
- Добавлен рейтинг и количество отзывов (не было в legacy)

## Производительность

- Используются индексы на полях `products.name` и `products.id`
- LEFT JOIN с `prices` для получения цен
- Подзапросы EXISTS для фильтрации (когда будет активирована)
- Лимит 10 результатов для автодополнения

## Известные ограничения

1. Фильтрация по авторам и возрастам подготовлена, но не активирована в UI
2. Нет сортировки результатов (всегда по названию ASC)
3. Нет поиска по описанию товара (только название и ID)
4. Автодополнение не учитывает наличие товара на складе
