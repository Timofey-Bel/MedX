# Реализация авторизации и профиля пользователя

## Обзор

Реализована полная система авторизации, регистрации и управления профилем пользователя для интернет-магазина "Сфера".

**Дата реализации:** 2024
**Статус:** ✅ Завершено

## Реализованные компоненты

### 1. AuthController

**Файл:** `app/Http/Controllers/AuthController.php`

**Методы:**
- `login()` - отображение формы входа
- `authenticate()` - обработка входа пользователя
- `register()` - отображение формы регистрации
- `store()` - обработка регистрации нового пользователя
- `logout()` - выход из системы

**Особенности:**
- Валидация данных с русскими сообщениями об ошибках
- Хеширование паролей через Hash::make()
- Автоматический вход после регистрации
- Перенаправление на страницу, с которой пришел пользователь (intended redirect)
- Использование прямых SQL-запросов для создания пользователя
- Проверка уникальности email

### 2. ProfileController

**Файл:** `app/Http/Controllers/ProfileController.php`

**Методы:**
- `index()` - отображение профиля и истории заказов
- `update()` - обновление данных пользователя
- `favorites()` - редирект на страницу избранного (deprecated)

**Особенности:**
- Отображение истории заказов через SQL-запрос
- Обновление имени, email и пароля
- Проверка текущего пароля при изменении
- Валидация уникальности email (исключая текущего пользователя)
- Использование прямых SQL-запросов для обновления данных

### 3. Blade шаблоны

#### Login (resources/views/auth/login.blade.php)
- Форма входа с полями email и password
- Чекбокс "Запомнить меня"
- Отображение ошибок валидации
- Ссылка на регистрацию
- CSRF-токен

#### Register (resources/views/auth/register.blade.php)
- Форма регистрации с полями: имя, email, пароль, подтверждение пароля
- Отображение ошибок валидации
- Подсказки для пользователя
- Ссылка на вход
- CSRF-токен

#### Profile (resources/views/profile/index.blade.php)
- Отображение данных пользователя
- Форма редактирования профиля
- Изменение пароля (опционально)
- История заказов с деталями
- Боковое меню с навигацией
- Кнопка выхода
- Адаптивный дизайн

## Роуты

### Публичные роуты
```php
Route::get('/login', [AuthController::class, 'login'])->name('login');
Route::post('/login', [AuthController::class, 'authenticate']);
Route::get('/register', [AuthController::class, 'register'])->name('register');
Route::post('/register', [AuthController::class, 'store']);
Route::post('/logout', [AuthController::class, 'logout'])->name('logout');
```

### Защищенные роуты (требуют авторизации)
```php
Route::middleware('auth')->group(function () {
    Route::get('/profile', [ProfileController::class, 'index'])->name('profile');
    Route::post('/profile/update', [ProfileController::class, 'update'])->name('profile.update');
    
    // Другие защищенные роуты
    Route::get('/checkout', [OrderController::class, 'checkout'])->name('checkout');
    Route::post('/checkout', [OrderController::class, 'placeOrder'])->name('checkout.place');

    Route::get('/orders', [OrderController::class, 'index'])->name('orders');
    Route::get('/order/{id}', [OrderController::class, 'show'])->name('order');
});
```

## Валидация

### Вход (login)
- `email` - обязательное поле, должен быть валидный email
- `password` - обязательное поле

### Регистрация (register)
- `name` - обязательное поле, максимум 255 символов
- `email` - обязательное поле, валидный email, уникальный в таблице users
- `password` - обязательное поле, минимум 6 символов, должен совпадать с подтверждением

### Обновление профиля (profile update)
- `name` - обязательное поле, максимум 255 символов
- `email` - обязательное поле, валидный email, уникальный (исключая текущего пользователя)
- `current_password` - обязательно при изменении пароля
- `password` - опционально, минимум 6 символов, должен совпадать с подтверждением

## Безопасность

### Реализованные меры безопасности:
1. **CSRF-защита** - все формы содержат @csrf токен
2. **Хеширование паролей** - использование Hash::make() и Hash::check()
3. **Валидация входных данных** - строгая валидация всех полей
4. **Параметризованные SQL-запросы** - защита от SQL-инъекций
5. **Middleware auth** - защита приватных страниц
6. **Session regeneration** - обновление сессии после входа/выхода
7. **Проверка текущего пароля** - при изменении пароля

## Структура базы данных

### Таблица users
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    email_verified_at TIMESTAMP NULL,
    password VARCHAR(255) NOT NULL,
    remember_token VARCHAR(100) NULL,
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL
);
```

### Таблица orders (для истории заказов)
```sql
CREATE TABLE orders (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    status VARCHAR(50) NOT NULL,
    total_amount DECIMAL(10,2) NOT NULL,
    delivery_address TEXT,
    phone VARCHAR(20),
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## Использование

### Регистрация нового пользователя
1. Перейти на `/register`
2. Заполнить форму (имя, email, пароль)
3. Нажать "Зарегистрироваться"
4. Автоматический вход и перенаправление в профиль

### Вход в систему
1. Перейти на `/login`
2. Ввести email и пароль
3. Опционально: поставить галочку "Запомнить меня"
4. Нажать "Войти"
5. Перенаправление на страницу, с которой пришел, или в профиль

### Управление профилем
1. Перейти в `/profile` (доступно только авторизованным)
2. Изменить имя или email
3. Опционально: изменить пароль (требуется текущий пароль)
4. Нажать "Сохранить изменения"
5. Просмотреть историю заказов

### Выход из системы
1. В профиле нажать кнопку "Выйти"
2. Или отправить POST-запрос на `/logout`
3. Перенаправление на главную страницу
