# Управление кешем Blade-шаблонов

## Проблема

Laravel компилирует Blade-шаблоны (`.blade.php`) в обычный PHP код и сохраняет их в `storage/framework/views/`. При открытии страницы Laravel использует **закешированную версию**, а не исходный файл.

**Симптомы:**
- Вы редактируете `.blade.php` файл
- Делаете `git commit`
- Открываете страницу в браузере
- **Изменения не отображаются** (показывается старая версия)
- Код есть в файлах, но не работает на сайте

## Почему это происходит?

1. Laravel компилирует Blade → PHP при первом обращении к странице
2. Сохраняет скомпилированный файл в `storage/framework/views/`
3. При следующих обращениях использует **закешированную версию**
4. Даже если вы изменили исходный `.blade.php`, Laravel продолжает использовать старый кеш

## Решения

### ✅ Решение 1: Автоматическая очистка после коммита (РЕКОМЕНДУЕТСЯ)

Создан Git hook `.git/hooks/post-commit`, который автоматически очищает кеш после каждого коммита.

**Как это работает:**
```bash
git commit -m "..."
# Автоматически выполняется:
# php artisan view:clear
```

**Преимущества:**
- Не нужно помнить про очистку кеша
- Работает автоматически после каждого коммита
- Гарантирует, что после коммита кеш всегда свежий

### ✅ Решение 2: Ручная очистка кеша

Если автоматический hook не сработал, очистите кеш вручную:

```powershell
& "C:\OS\modules\PHP-8.5\php.exe" artisan view:clear
```

**Когда использовать:**
- После редактирования любого `.blade.php` файла
- Перед тестированием изменений
- Если страница показывает старую версию

### ✅ Решение 3: Очистка всего кеша приложения

Если проблема не только с view, очистите весь кеш:

```powershell
& "C:\OS\modules\PHP-8.5\php.exe" artisan cache:clear
& "C:\OS\modules\PHP-8.5\php.exe" artisan config:clear
& "C:\OS\modules\PHP-8.5\php.exe" artisan route:clear
& "C:\OS\modules\PHP-8.5\php.exe" artisan view:clear
```

## Правило для разработки

### КРИТИЧНО: Всегда очищайте кеш после изменения Blade-шаблонов

**Workflow:**
1. Редактируете `.blade.php` файл
2. Сохраняете изменения
3. **Очищаете кеш:** `php artisan view:clear`
4. Обновляете страницу в браузере
5. Проверяете изменения
6. Делаете `git commit` (кеш очистится автоматически)

## Файлы, которые требуют очистки кеша

**Всегда очищайте кеш после изменения:**
- `resources/views/**/*.blade.php` - все Blade-шаблоны
- `resources/views/components/**/*.blade.php` - Blade-компоненты
- `resources/views/layouts/**/*.blade.php` - layouts

**Не требуют очистки кеша:**
- `public/assets/**/*.css` - CSS файлы (обновляются сразу)
- `public/assets/**/*.js` - JavaScript файлы (обновляются сразу)
- `app/Http/Controllers/**/*.php` - контроллеры (обновляются сразу)
- `routes/web.php` - маршруты (требуют `route:clear`, но не `view:clear`)

## Отладка проблем с кешем

### Проблема: Изменения не отображаются

**Шаги:**
1. Проверьте, что файл сохранён: `git status`
2. Очистите кеш: `php artisan view:clear`
3. Обновите страницу с очисткой кеша браузера: `Ctrl+F5`
4. Проверьте консоль браузера на ошибки: `F12`

### Проблема: Ошибка "View not found"

**Причина:** Кеш содержит ссылку на несуществующий файл

**Решение:**
```powershell
& "C:\OS\modules\PHP-8.5\php.exe" artisan view:clear
```

### Проблема: Синтаксическая ошибка в Blade

**Причина:** Blade скомпилировался с ошибкой, кеш содержит неправильный код

**Решение:**
1. Исправьте синтаксическую ошибку в `.blade.php`
2. Очистите кеш: `php artisan view:clear`
3. Обновите страницу

## Git и кеш

### Что НЕ нужно коммитить

**НИКОГДА не добавляйте в Git:**
```
storage/framework/views/*.php
```

Эти файлы генерируются автоматически и должны быть в `.gitignore`.

**Перед коммитом всегда сбрасывайте изменения:**
```powershell
git checkout -- storage/framework/views/
```

### Что нужно коммитить

**Всегда коммитьте:**
- `resources/views/**/*.blade.php` - исходные Blade-шаблоны
- `public/assets/**/*.css` - стили
- `public/assets/**/*.js` - скрипты
- `app/Http/Controllers/**/*.php` - контроллеры

## Чек-лист перед коммитом

- [ ] Очистили кеш: `php artisan view:clear`

- [ ] Сбросили изменения в `storage/framework/views/`: `git checkout -- storage/framework/views/`

- [ ] Сделали коммит: `git commit -m "..."`
