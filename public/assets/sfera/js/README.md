# JavaScript файлы сайта

## Описание

Эта папка содержит все JavaScript файлы для фронтенда сайта. Файлы организованы по функциональности и используют различные подходы: нативный JavaScript, KnockoutJS для реактивных интерфейсов, и AJAX для взаимодействия с сервером.

---

## Отладка через консоль браузера

Все ViewModels доступны глобально для отладки через консоль браузера на любой странице:

### Корзина

```javascript
// Короткие алиасы
cart                    // или cartViewModel
favorites               // или favoritesViewModel

// Методы отладки
cart.debugInfo()                    // Вывести полную информацию о корзине в консоль
cart.getCartState()                 // Получить состояние корзины как объект
cart.getItemById('123')             // Получить товар по ID (возвращает CartItem)
cart.debugItem('123')               // Вывести подробную информацию о товаре в консоль

// Прямой доступ к данным
cart.items()                        // Массив товаров (observableArray)
cart.itemsTotal()                   // Сумма товаров (computed)
cart.grandTotal()                   // Итоговая сумма с доставкой и скидками (computed)
cart.promoCode()                    // Промокод (observable)
cart.deliveryPrice()                 // Цена доставки (observable)
```

### Избранное

```javascript
// Методы отладки
favorites.debugInfo()               // Вывести полную информацию об избранном в консоль
favorites.getFavoritesState()       // Получить состояние избранного как объект
favorites.getItemById('123')        // Получить товар по ID (возвращает FavoriteItem)
favorites.debugItem('123')          // Вывести подробную информацию о товаре в консоль

// Прямой доступ к данным
favorites.items()                   // Массив товаров (observableArray)
favorites.itemsCount()              // Количество товаров (computed)
```

### Примеры использования

```javascript
// Просмотр состояния корзины
model_cart.debugCart()

// Просмотр количества товаров
console.log('Товаров в корзине:', model_cart.items().length)

// Просмотр суммы
console.log('Сумма товаров:', model_cart.cart_sum(), '₽')
console.log('Итого с учетом скидки:', model_cart.total_cart_sum(), '₽')

// Просмотр всех товаров в корзине
model_cart.items().forEach(function(item, index) {
    console.log(index, item.id(), item.name(), item.formattedPrice(), 'x' + item.product_amount())
})

// Получение объекта items (ключ-значение)
var itemsObj = model_cart.getItemsObject()
console.log('Товары как объект:', itemsObj)

// Проверка промокода
if (model_cart.promocode_applied()) {
    console.log('Промокод применен:', model_cart.promocode(), 'Скидка:', model_cart.cart_discount(), '₽')
}

// Просмотр данных корзины
var cartData = model_cart.cart()
console.log('Данные корзины:', cartData)
console.log('Items в cart:', ko.unwrap(cartData.items))
```

**Примечание:** Модель создается глобально при загрузке скрипта `model_cart.js.tpl`, поэтому доступна на любой странице сайта через `model_cart` или `window.model_cart`. Также доступен короткий алиас `cart` через `window.cart = model_cart`.

### 1. `cart-viewmodel.js` (39KB, 986 строк)

**Назначение:** Главная KnockoutJS ViewModel для управления корзиной покупок (legacy, используется в некоторых местах).

**Примечание:** В текущем проекте используется модульный подход с `site/modules/models/cart_new/model_cart.js.tpl` и модульными методами. Этот файл может использоваться для обратной совместимости или в других частях проекта.

**Основные функции:**
- **MVVM паттерн:** Реактивное управление состоянием корзины через KnockoutJS
- **CartItem модель:** Представление товара в корзине с observables для количества, цены, скидок
- **Вычисляемые свойства:** `formattedPrice`, `formattedTotal`, `formattedQuantity`, `discountTotal`, `grandTotal`
- **AJAX операции:**
  - `addItem()` - добавление товара в корзину
  - `removeItem()` - удаление товара
  - `updateItemQuantity()` - обновление количества
  - `applyPromoCode()` - применение промокода
  - `removePromoCode()` - удаление промокода
  - `clearCart()` - очистка корзины
- **Интеграция с checkout:** Поддержка доставки, промокодов, динамическое обновление итогов
- **Уведомления:** Система уведомлений о действиях с корзиной

**Использование:**
- Может использоваться в некоторых частях проекта для обратной совместимости
- Основная реализация находится в `site/modules/models/cart_new/`

## Архитектурные паттерны

### MVVM (Model-View-ViewModel)

Используется в файлах:
- `cart-viewmodel.js` - управление корзиной
- `favorites-viewmodel.js` - управление избранным
- `cart-counter.js` - счетчик корзины
- `favorites-counter.js` - счетчик избранного

**Преимущества:**
- Реактивное обновление UI при изменении данных
- Разделение логики и представления
- Легкое тестирование

### AJAX взаимодействие

Все файлы используют `fetch()` API для:
- Добавления/удаления товаров в корзину
- Управления избранным
- Применения промокодов
- Поиска товаров и авторов

### Обратная совместимость

Файлы `cart.js` и `favorites.js` проверяют наличие KnockoutJS и пропускают инициализацию, если используется ViewModel подход.

## Зависимости

### Внешние библиотеки:
- **KnockoutJS** - для реактивных интерфейсов (cart-viewmodel, favorites-viewmodel, счетчики)
- **jQuery** - используется в некоторых местах (например, в checkout.tpl для Yandex Maps)

### Внутренние зависимости:
- `cart-viewmodel.js` используется в `checkout.js` для обновления итогов

- Все файлы могут использовать глобальную функцию `showNotification()` (определяется в каждом файле)

## Особенности реализации

### 1. Синхронизация состояния

- **Корзина:** Состояние синхронизируется через AJAX запросы и обновляется в реальном времени
- **Избранное:** Аналогично корзине, с реактивным обновлением через KnockoutJS
- **Счетчики:** Автоматически обновляются при любых действиях с корзиной/избранным

### 2. Обработка ошибок

- Все AJAX запросы имеют `.catch()` обработчики
- При ошибках показываются уведомления пользователю
- Логирование в консоль для отладки

### 3. Производительность

- **Debounce:** Поиск и автодополнение используют задержку (300ms)
- **Lazy loading:** Некоторые функции инициализируются только при необходимости
- **Event delegation:** Где возможно, используется делегирование событий

### 4. Адаптивность

- Проверка размера экрана для мобильных/десктоп версий
- Перемещение элементов на мобильных устройствах (например, sidebar на странице товара)
- Touch события для мобильных устройств (swipe в слайдерах)

---

## Рекомендации по использованию

1. **Для новых страниц:** Используйте KnockoutJS ViewModels для реактивных интерфейсов
2. **Для простых страниц:** Достаточно нативного JavaScript
3. **Для счетчиков:** Всегда используйте глобальные ViewModels (`cartCounterViewModel`, `favoritesCounterViewModel`)
4. **Для уведомлений:** Используйте функцию `showNotification()` из соответствующих файлов или создайте общую
