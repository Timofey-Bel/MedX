# Тесты функционала корзины (CartFunctionalityTest)

## Описание

Комплексный набор из 29 тестов для проверки всех аспектов функционала корзины покупок.

## Покрытие тестами

### 1. Добавление товара в корзину (3 теста)
- ✅ Добавление нового товара
- ✅ Увеличение количества существующего товара
- ✅ Проверка структуры ответа

### 2. Удаление товара из корзины (3 теста)
- ✅ Удаление существующего товара
- ✅ Попытка удалить несуществующий товар
- ✅ Удаление одного товара при наличии нескольких

### 3. Изменение количества (4 теста)
- ✅ Увеличение количества
- ✅ Уменьшение количества
- ✅ Установка количества в 0 (удаление товара)
- ✅ Установка отрицательного количества (удаление товара)

### 4. Функционал selected (галочка) (6 тестов)
- ✅ Снятие галочки с товара
- ✅ Пересчет суммы без невыбранного товара
- ✅ Пересчет количества без невыбранного товара
- ✅ Установка галочки обратно
- ✅ Включение товара в суммы после установки галочки
- ✅ Работа с несколькими товарами с разными состояниями selected

### 5. Подсчет итоговых сумм (5 тестов)
- ✅ cart_sum учитывает только selected товары
- ✅ total_cart_amount учитывает только selected товары
- ✅ total_cart_sum = cart_sum - cart_discount
- ✅ Пустая корзина возвращает нулевые суммы
- ✅ Корзина со всеми невыбранными товарами возвращает нулевые суммы

### 6. Очистка корзины (3 теста)
- ✅ Полная очистка корзины
- ✅ Очистка состояния cart_selected
- ✅ Очистка промокода и скидки

### 7. Сессионное хранение (5 тестов)
- ✅ Данные корзины сохраняются в сессии
- ✅ cart_selected хранится отдельно
- ✅ Данные сохраняются при множественных операциях
- ✅ Новые товары выбраны по умолчанию
- ✅ Формат сессии соответствует ожидаемому

## Требования

### База данных
Тесты работают с **реальной БД проекта** (не in-memory SQLite).

Убедитесь что:
1. В БД есть минимум 3 товара в таблице `products`
2. Для товаров есть цены в таблице `prices` с `price_type_id = '000000002'`

### Конфигурация тестов

Если тесты используют SQLite in-memory БД, измените конфигурацию в `phpunit.xml`:

```xml
<php>
    <!-- Используйте реальную БД вместо :memory: -->
    <env name="DB_CONNECTION" value="mysql"/>
    <env name="DB_DATABASE" value="your_test_database"/>
</php>
```

Или создайте отдельный файл `.env.testing`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_test_database
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

### Запуск всех тестов корзины
```bash
php artisan test --filter=CartFunctionalityTest
```

### Запуск конкретного теста
```bash
php artisan test --filter=it_can_add_new_item_to_cart
```

### Запуск с остановкой на первой ошибке
```bash
php artisan test --filter=CartFunctionalityTest --stop-on-failure
```

### Запуск с подробным выводом
```bash
php artisan test --filter=CartFunctionalityTest --verbose
```

## Структура тестов

Каждый тест следует паттерну AAA (Arrange-Act-Assert):

```php
/** @test */
public function it_can_add_new_item_to_cart()
{
    // Arrange (Подготовка)
    // Тестовые данные загружаются автоматически в setUp()
    
    // Act (Действие)
    $result = $this->cartService->addItem($this->testProductId1, 2);
    
    // Assert (Проверка)
    $this->assertArrayHasKey($this->testProductId1, $result['items']);
    $this->assertEquals(2, $result['items'][$this->testProductId1]['product_amount']);
}
```

## Примечания

1. **Автоматическая загрузка товаров**: Тесты автоматически загружают первые 3 товара из БД в методе `setUp()`

2. **Очистка сессии**: Перед каждым тестом сессия очищается для изоляции тестов

3. **Пропуск тестов**: Если в БД меньше 3 товаров, тесты будут пропущены с соответствующим сообщением

4. **Комментарии на русском**: Все тесты содержат подробные комментарии на русском языке

## Устранение проблем

### Ошибка "no such table: products"
Тесты пытаются использовать in-memory SQLite БД. Решение:
- Настройте использование реальной БД в `phpunit.xml` или `.env.testing`

### Ошибка "Недостаточно товаров в БД"
В БД меньше 3 товаров. Решение:
- Добавьте товары в БД через seeder или вручную

### Тесты падают из-за отсутствия цен
Для товаров нет цен в таблице `prices`. Решение:
- Убедитесь что для каждого товара есть запись в `prices` с `price_type_id = '000000002'`
